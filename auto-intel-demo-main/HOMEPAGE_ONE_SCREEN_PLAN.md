# 首页一屏优化方案

## 目标

- 首页必须在常见笔记本视口内完成一屏展示，不依赖长滚动。
- 首页只负责“给判断”，不承担完整证据展开。
- 首页默认面向中文客户，外文事件继续默认展示中文译文，并明确标注“原文非中文”。
- 首页必须清楚说明“本期早报窗口”，避免再回到自然日语义。

## 信息架构

首页拆成 3 层。

### 1. 顶部状态条

用途：先把这期早报到底覆盖什么范围讲清楚。

显示内容：
- 期次日期
- 早报窗口起止时间
- 截止时间
- 最近更新时间
- 最近任务状态

设计要求：
- 单行、强信息密度
- 纯中文文案
- 不放解释性长句

### 2. 主判断区

用途：客户第一眼看到“今天该讲什么”。

左侧大卡：
- 模块名：`今日主线`
- 内容长度控制在 80-120 字
- 固定表达 3 件事：
  - 主线是什么
  - 为什么重要
  - 下一步该盯什么

右侧压缩指标：
- 本期资讯数
- 候选事件数
- 重大事件数
- 境内 / 境外重大事件

设计要求：
- 左重右轻
- 数字卡压缩成 4 个
- 不再重复展示大段解释

### 3. 证据区

用途：用最少空间证明首页判断不是拍脑袋。

左列：`头部事件`
- 只放 3 条
- 每条只保留：
  - 标题
  - 来源
  - 分类
  - 重要度
  - 一句话摘要
  - `原文非中文` 标记

中列：`持续跟踪`
- 只放 2 条
- 明确标识这是前几期延续项，不伪装成当期新增
- 每条只显示标题和来源

右列：`系统在线状态`
- 顶部显示：
  - 运行成功数
  - 失败数
  - 运行中数
- 下方用 9 个来源微卡或状态点阵展示来源在线情况
- 不展示完整运行表格

## 明确移出首页的内容

以下内容不再在首页完整展示：
- 完整来源覆盖列表
- 完整风险 / 机会长列表
- 完整运行明细表
- 超过 3 条的头部事件
- 长篇解释性摘要

这些内容分别保留在：
- `/intel`
- `/ops`

## 视觉方向

定位为“中文财经战情板”，不是常规 dashboard 卡片墙。

设计原则：
- 顶部状态条像情报终端状态栏
- 主判断区有明确主视觉中心
- 下半区密度高但结构稳定
- 颜色继续沿用当前暖色系，但减少大面积重复卡片感
- 首页尽量避免英文模块名

## 页面约束

- 导航高度固定
- 主判断区与证据区总高度控制在一屏内
- 头部事件和来源状态区不能无限增高
- 发生内容溢出时，优先裁剪条目数量，不允许首页无限变长

## 需要的上下文字段

在现有首页 payload 基础上补这些只给模板使用的字段：
- `source_status`
- `source_health`
  - `success_count`
  - `failed_count`
  - `running_count`
  - `idle_count`
- `hero_event`
  - 默认取首条头部事件

## 验收标准

- 首页在 `http://127.0.0.1:8018/` 下可一屏看完核心内容
- 首屏不需要滚动即可看到：
  - 早报窗口
  - 今日主线
  - 3 条头部事件
  - 2 条持续跟踪
  - 系统状态摘要
- 首页全部使用中文模块名
- 外文事件仍默认显示中文译文，并明确显示 `原文非中文`
- `/intel` 和 `/ops` 不受影响
